If you notice on my design webpage, I include a site called moodboard project. What might surprise you about this website is knowing that the image gallery itself is generated by Javascript. The code that I used to do this is pasted below: function img(img) {
var backgroundImage;
var hold;
// creates div that will contain image
backgroundImage = document.createElement("div");
//sets the background image to "1.jpg", "2.jpg" etc. (if you use a folder other than images, change the phrase in the code below and on line 13)
backgroundImage.setAttribute("style","background-image: url(\"images/" + img + ".jpg\");");
backgroundImage.setAttribute("id",img);
backgroundImage.setAttribute("class","img");
// makes the image holding div
hold = document.createElement("button");
hold.setAttribute("class","hold");
hold.setAttribute("onclick","Viewing(\"images/" + img + ".jpg\");");
// nests the div
hold.appendChild(backgroundImage);
//‘gallery’ is the div for the images to be created into
document.getElementsByClassName('gallery')[0].appendChild(hold); };
//some examples of usage
img("bricks");
img("houses");
img("leaf");
img("board");
img("kiwi");
img("lizard”);
 It’s important to note that the frame and border are styled due to css. To imitate the styling that I use on my moodboard, add an outset border to buttons with the id “hold”.
I originally wanted to have the subheadings adapt to when the browser was resized instead of the main text divs. I ended up on a solution which is talked in more detail at the About section of the website. I also originally wanted to try to integrate syntax highlighting into snippets of code on the website, but I found that it made the text less readable.
